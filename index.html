Summary

We aim to develop a fast Distributed GPU ray tracer that has the following features:

  1. Very Fast Ray Traversal
    Even for scenes with 100K+ triangles, our ray tracer can achieve a throughput of 200+ Mrays/s on GTX 680 for primary ray. And we can render a 1000x1000 picture of the scene with ambient occlusion at 60 FPS, while its CPU counterpart stays with 0.30 FPS.
    We made a lot of efforts to improve the efficiency of ray traversal:

    • We used persistent threads to achieve better SIMD utilization.
    • We used per-ray stack for ray traversal.
    • We used while-while structure for node traversal and primitive intersection test.
    • We optimized ray-triangle intersection and ray-box intersection.
    • We also tried out speculative method, i.e., delay the primitive intersection test until all lanes find a leaf. After all these optimizations, over ray tracer is 20x faster than original one.

  2. Very Fast Parallel BVH build
    We can build BVH of scenes with 240K+ triangles in 8 ms, while the CPU build takes nearly 2 second. We use Morton codes to encode each primitive such that we can know the exact position of each node in the array. Therefore, we can build the BVH tree in fully parallel.

  3. Distributed Ray Tracing across Clusters We implemented a server client mechanism for the system, which allows us to render large scenes with multiple machines and GPUs.

  4. Versatile
    Our ray tracer support almost all the functionalities of its CPU counterpart:

    • Monte Carlo sampling
    • Multiple types of BSDFs: refraction, Fresnel reflection, e.g.
