Project Proposal

Summary

The project aims at accelerating the computationally intensive ray tracing algorithm. To achieve this end, we will utilize the computing power of both GPU and distributed machines. We are going to build BVH on GPU to allow faster ray-triangle intersection test. For machines on clusters, they all build the same BVH but will process different parts of the image. We will treat patches of the image as jobs and use some workload balance strategy.

Background

Ray tracing is one of the most researched fields in Computer Graphics. It is a great technique to produce optical effects such as reflection, refraction, scattering and dispersion phenomena. Its capability of producing a very high degree of visual realism, which is usually higher than that of typical scanline rendering methods, comes with a greater computational cost. So most of the research is focused on how to make it more time efficient. There are many ways to achieve this goal. In this project, we will focus on taking advantage of parallel computing power.

One way to make ray tracing faster is to use GPU. This is due to the parallel nature of ray tracing. The most computationally intensive part of ray tracing is the ray-primitive intersection test. Intersection tests of different rays are independent of one another, which leaves much room for parallel acceleration. GPU provide huge amount of parallel executing threads to exploit the parallelism in ray tracing.

Compared to network latency, the computation time of ray tracing is much higher. So it hints us to use distributed machines to accelerate ray tracing. Different patches of the image are independent of one another. So we treat a single patch as a unit of work, and send the work to different machines using load balancing algorithm.

The challenge

There are several challenges in this project:

  1. Constructing and traversing BVH on GPU is much more complex than CPU because GPU is not good at logic control.
  2. Debugging and tuning GPU code is much trickier. It maybe easy to implement a GPU version that has 4x speedup, but it's hard to make it achieve 20x speedup.
  3. Different machines need to communicate to be set up with the same configuration. And they need to cooperate with each other to get the final solution. Implementing these communicating interfaces can be time consuming.

Resources

We are using the path tracer code of 15-462 assignment as the start code, its CPU version is done by Ye Yuan last semester. We will refer to multiple papers, which will be in the reference section.

Goals and Deliverables

Goals to achieve

  1. Implement fast GPU-based ray tracing.
  2. Implement distributed ray tracing.

Extra goals

  1. Achieve near real time ray tracing on complex scenes (5-10 fps).

Deliverables

  1. Images showing rendering results.
  2. Speedup graphs.

Platform Choice

We will be using C++ as programming language because it's most convenient for CUDA programming. And we will develop the program on Linux and OS X, so we can potentially use clusters as distributed machines.

Schedule

  • Week1: project proposal, literature review, choosing methods to use.
  • Week2-3: implement GPU-based ray tracing.
  • Week4-5: implement distributed ray tracing.
  • Week6: make results and write final report.
